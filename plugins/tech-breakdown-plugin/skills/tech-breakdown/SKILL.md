---
description: Learn any technology through a structured 3-part breakdown — what it is, why it exists, and how it works at its core
---

The user wants to learn about a technology. If they haven't specified one, ask them which technology they want to learn.

Produce a structured breakdown in the following format. Write in the same language the user used.

---

# [Technology Name]

## 1. 是什么 (What)

Give a single crisp paragraph (3–5 sentences) that defines the technology. Avoid jargon. A complete beginner should finish this section with a clear mental model of what it is.

## 2. 为什么 (Why)

Explain the problem that existed before this technology, and why existing solutions were insufficient. Describe the historical or technical context that motivated its creation. Keep it to 3–5 sentences — the goal is to make the reader feel the pain this technology solves.

## 3. 怎么做 (How)

This is the core of the breakdown. Structure it as:

### 核心组成

List the 3–6 key components or concepts that make up this technology. For each one, give a one-line description. Bold the component name.

### 工作原理

Explain how the pieces fit together and how the technology works end-to-end. Use an ASCII diagram whenever it helps illustrate flow, architecture, or structure. For example:

```
[Input] → [Step A] → [Step B] → [Output]
```

Or for layered architectures:

```
┌─────────────────────┐
│      Layer N        │
├─────────────────────┤
│      Layer N-1      │
└─────────────────────┘
```

Use diagrams liberally — if a relationship or flow can be drawn, draw it.

### 核心记忆点

End with a single bold sentence or short list (≤3 items) that captures the essential insight a learner must remember. This should be memorable enough to recall a week later.

---

After the breakdown, ask: "想深入了解哪个部分，或者想看一个具体的例子吗？"
